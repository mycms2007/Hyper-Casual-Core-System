
## 언어 및 응답 형식

- 모든 응답은 한국어로
- 수정 지시는 반드시 단계별 번호로 작성
- 각 단계는 1~2줄 이내로 간결하게
- Unity 에디터 작업은 "어디 선택 → 뭐 클릭 → 뭐 입력" 형식으로
- 새 스크립트를 추가하거나 씬 구성이 바뀌는 작업 완료 후:
  ### 씬별 필요 오브젝트 목록
  ### Inspector 연결 목록 (슬롯명 | 연결할 오브젝트)

# 작업 워크플로우

모든 구현 작업은 아래 순서를 반드시 따른다.

## 1. 코드 리서치
- 관련 파일을 읽고 현재 구조를 파악한다
- 수정이 필요한 범위와 영향 받는 파일을 확인한다

## 2. 계획 (plan.md)
- `plan.md`에 구현 방식, 트레이드오프, 파일 목록을 정리한다
- 사용자의 리뷰/승인을 받은 후 다음 단계로 넘어간다

## 3. 주석 달기
- 구현 전, 변경할 위치에 `// TODO: ...` 형태로 무엇을 어떻게 바꿀지 먼저 표시한다
- 사용자가 확인 후 다음 단계로 넘어간다

## 4. 구현
- 승인된 계획대로 코드를 작성한다
- 계획 외의 변경은 하지 않는다
- 코드 작성 시 `andrej-karpathy-skills:karpathy-guidelines` 스킬을 적용한다

## 5. 피드백
- 구현 완료 후 사용자의 피드백을 기다린다
- 피드백은 `plan.md`에 리뷰로 달리며, 요청 시 확인 후 반영한다
